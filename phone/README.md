# 원무 전화응대 도우미 📞

병·의원 **원무 데스크의 전화 부담을 줄이기 위한** 웹 도구입니다.
부재중 전화를 놓치지 않게 모아 관리하고, 자주 하는 안내를 빠르게 찾고,
문자를 한 번에 보낼 수 있게 도와줍니다.

- 설치 불필요 · 서버 불필요 — 브라우저만 있으면 됩니다.
- **모든 기록은 사용하는 기기(브라우저)에만 저장**됩니다. 환자 번호·메모가 외부로 전송되지 않습니다.
- PC·태블릿·휴대폰 모두 지원 (반응형).

배포 주소(예): `https://yandi63-source.github.io/chocum-cherom-data/phone/`

---

## 왜 만들었나 (현재 문제)

- KT 인터넷 전화기 사용 중 **부재중 전화가 많음** → 콜백을 놓침
- 원무 직원이 반복 문의(진료시간·검사 준비·주차 등) 응대와 문자 작성으로 **업무 부하가 큼**

이 도구는 위 두 가지를 바로 줄여 줍니다.

---

## 기능

### 1) 부재중 콜백 보드
- 부재중 번호를 한 건씩 등록하거나, 통화기록을 **붙여넣어 여러 건 자동 등록**(번호 자동 추출).
- 각 항목에서 **📞 전화 · ✉️ 문자**를 한 번에 실행 (모바일·태블릿에서 바로 통화/문자앱 연결).
- 상태 관리: **미처리 → 완료 / 보류**. 급함 표시로 우선순위 정리.
- 미처리 건수는 상단에 항상 표시되어 놓치지 않습니다.

### 2) 응대 스크립트 (FAQ)
- 진료시간·예약·검사 준비(내시경/혈액/초음파)·결과·서류·비용 등 **자주 묻는 질문의 표준 답변**.
- 검색으로 즉시 찾고, "📋 안내문 복사"로 문자에 붙여넣기.
- **✏️ 편집** 모드에서 우리 병원에 맞게 자유롭게 수정·추가·삭제.

### 3) 문자 템플릿
- 부재중 회신, 예약 확인/알림, 내시경 준비, 결과 안내, 증명서 안내 등.
- 상단에 **환자명·날짜·시간**을 입력하면 템플릿의 `{환자명}` `{날짜}` `{시간}`이 자동으로 채워집니다.
- `{병원명}` `{대표전화}`는 설정에 입력한 병원 정보로 자동 치환.
- "📋 복사" 또는 "✉️ 문자앱 열기"로 바로 발송.

### 4) 설정
- 병원 이름·대표 전화 입력(문자 서명에 사용).
- 화면 테마(밝게/어둡게/자동).
- **백업 내보내기/불러오기** — 기기를 바꾸거나 여러 대에서 같은 내용을 쓰려면 JSON 파일로 백업.
- 완료 콜백 정리 · 전체 초기화.

## 처음 설정 (권장 순서)

1. **설정** 탭에서 병원 이름·대표 전화를 입력하고 저장.
2. **응대 스크립트** 탭에서 **✏️ 편집**을 눌러 진료시간·주차·위치·검사 준비 안내를 우리 병원 내용으로 수정.
3. **문자 템플릿** 탭에서 문구를 다듬기.
4. 데스크 PC·태블릿의 브라우저에 **즐겨찾기/홈 화면 추가**(PWA 설치)해서 앱처럼 사용.
5. 여러 기기에서 같은 내용을 쓰려면 한 기기에서 **백업 내보내기** → 다른 기기에서 **불러오기**.

> ⚠️ 데이터는 기기별로 따로 저장됩니다. 여러 직원이 각자 기기에서 쓰면 콜백 목록이 공유되지 않습니다.
> 콜백 목록을 실시간 공유해야 한다면 아래 "다음 단계"의 공용 서버 연동이 필요합니다.

## 다음 단계 (KT 인터넷 전화 자동 연동 등)

지금 버전은 **직원이 부재중 번호를 등록/붙여넣는 반자동 방식**입니다.
아래는 더 자동화하려면 필요한 작업과 현실적인 방법입니다.

1. **부재중 전화 자동 수집**
   - KT 인터넷 전화(예: 스마트벨링/오피스 등) 상품에 따라 **통화기록 API·CDR 연동**이나 관리자 페이지 내보내기가 가능한 경우가 있습니다. KT 고객센터/기업영업에 "통화기록(CDR) 조회 API 제공 여부"를 문의해야 합니다.
   - 대안: **착신전환 + 클라우드 전화(예: 가상번호/콜센터형 서비스)**로 바꾸면 부재중 이벤트를 웹훅으로 받아 이 보드에 자동 등록할 수 있습니다.

2. **자동응답(ARS/AI 상담)** — 전화량 자체 줄이기
   - 진료시간·위치·예약 같은 반복 문의는 ARS 안내나 AI 음성 응답으로 처리 가능. 클라우드 전화 서비스 또는 별도 음성봇 연동이 필요합니다.

3. **부재중 자동 회신 문자(콜백 SMS)**
   - 부재중 발생 시 "확인 후 연락드립니다" 문자를 자동 발송. 문자 발송 API(정보통신망법상 사전 동의·야간 발송 규정 준수) 연동이 필요합니다.

4. **온라인 예약/카카오 채널 전환**
   - 전화 대신 웹 예약·카카오톡 상담으로 유도해 전화량을 구조적으로 줄이는 방법.

5. **콜백 목록 실시간 공유(다중 기기)**
   - 여러 데스크가 같은 콜백 목록을 보려면 간단한 공용 백엔드(예: 구글 시트/파이어베이스/작은 서버)가 필요합니다.

> 위 항목들은 통신사 상품·개인정보/광고문자 관련 법규 확인이 선행되어야 합니다.
> 어떤 방향을 원하시는지 알려주시면 해당 부분을 이어서 개발하겠습니다.
